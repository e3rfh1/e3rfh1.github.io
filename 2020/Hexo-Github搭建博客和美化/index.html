3.Hexo博客搭建

3.1 简介

  • Hexo(中文官方网站)是一个快速, 简洁且高效的博客框架. 让上百个页面在几秒内瞬间完成渲染. Hexo支持Github Flavored Markdown的所有功能, 甚至可以整合Octopress的大多数插件. 并自己也拥有强大的插件系统.

    3.2 配置SSH密钥与安装Hexo

    确保你已下载并安装git,安装完成后打开 Git Bash(以下命令皆在Git Bash使用)
    首先先配置SSH key 如果提示：No such file or directory 说明你是第一次使用git。
    接着输入 然后连续3次回车，最终会生成一个文件在用户目录下，打开用户目录，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key：

将刚复制的内容粘贴到key那里，title随便填，保存。

3.3 测试是否成功

$ ssh -T git@github.com # 注意邮箱地址不用改

如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到：

Hi XXX! You've successfully authenticated, but GitHub does not provide shell access.

看到这个信息说明SSH已配置成功！

4.5 上传之前

在上传代码到github之前，一定要记得先把你以前所有代码下载下来（虽然github有版本管理，但备份一下总是好的），因为从hexo提交代码时会把你以前的所有代码都删掉。

5. 美化

5.1 github图床搭建

  • 适用环境：win + mac + linux
  • 需要工具：GitHub 账号 + PicGo 客户端
  • 稳定性：背靠 GitHub 和微软，比自建服务器都稳
  • 隐私性：这算是唯一缺点，你的图片别人可以访问

5.1.1 GitHub 仓库设置

流程：新建 public 仓库 -> 创建 token -> 复制 token 备用

新建仓库

点击 git 主页右上角的+创建 New repository；

具体流程跟创建博客差不多一样，但区别在填写仓库信息

填写仓库信息，例如我就创建了一个 cloudimg 的仓库。这里注意，仓库得设置为 Public 因为后面通过客户端访问算是外部访问，因此无法访问 Private ，这样的话图片传上来之后只能存储不能显示。所以要设置为 Public。

创建 token 并复制保存
